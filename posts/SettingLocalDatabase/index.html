Virtualbox 설치

_1. Homebrew를 이용하여 Virtualbox 설치

1

brew install --cask virtualbox


Virtualbox에 Ubuntu 설치

_1. Ubuntu 공식 홈페이지를 통한 OS 이미지 다운로드

  • 공식 홈페이지

_2. Ubuntu 이미지를 Virtualbox에 올림

  • 새 OS 생성 시 다운받아놓은 Ubuntu 이미지 불러옴

  • OS 계정의 아이디와 비밀번호 설정

  • OS의 메모리와 CPU 할당

  • 하드디스크 용량 할당

  • 설정한 값대로 OS 생성


_3. Terminal이 열리지 않을 때

  • 언어 설정을 바꿔줌
  • English(United States) -> English(United Kingdom)으로 바꿔줌

  • 왜?


Ubuntu 서버에 Docker 설치

_1. 공식문서 참고

  • 공식 홈페이지

_2. apt 패키지를 업데이트하고, apt 패키지가 https로 repository를 사용할 수 있도록 패키지 다운로드

1
2
3
4
5
6

sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

_3. Docker 공식 GPG key를 다운로드 (보충 필요)

1
2

sudo mkdir -m 0755 -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

_4. 아래 명령어로 repository를 다운로드 (보충 필요)

1
2
3

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

_5. Docker 엔진을 내려받기 전 apt 패키지를 업데이트

1

sudo apt-get update

_6. Docker 엔진 등 필요 플러그인 설치

1

sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

_7. 아래 명령어로 docker가 정상적으로 설치되었는지 확인

1

sudo docker run hello-world


Mysql Container 띄우기

_1. Dockerhub에서 Mysql Image 내려받기

1

docker pull mysql:latest


_2. 내려받은 Mysql Image로 Container 생성 및 실행

1
2
3
4
5

docker run --name mysql -d \ // 컨테이너를 백그라운드에서 실행
  -p 3306:3306 \ // host의 3306 포트와 컨테이너의 3306 포트를 매핑
  -e MYSQL_ROOT_PASSWORD=<password> \ // root 사용자의 비밀번호를 지정
  --restart unless-stopped \ // 컨테이너가 비정상적으로 종료될 경우 재시작
  mysql:latest // 최신버전의 이미지를 사용하여 컨테이서 생성


_3. Host OS와 VM 네트워크 연결 - Port Fowarding 처리

  • VM의 IP를 조회하면 10.0.2.15가 나옴.

  • Host IP, Host Port 번호로 네트워크 요청이 들어올 경우, VM의 IP, Port로 포워딩을 해줌.


_4. Host OS에서 MySQL 연결

  • 연결 시 Driver properties에서 allowPublicKeyRetrieval = true로 설정. (Default는 false)

  • MySQL 8.0버전 이상에서부터 생길 수 있음.

  • 정상 연결 확인


=> MySQL Local Database 셋팅이 완료됨.
